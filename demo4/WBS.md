# 工作分解结构 (WBS) — 电商购物平台项目

## 项目名称：基于 Spring Boot 的电商购物平台

## 项目概述
本项目是一个全功能的电子商务购物平台，采用 Spring Boot + MySQL + JPA + 纯前端 HTML/CSS/JS 技术栈，支持用户注册登录、商品浏览与管理、购物车、订单全生命周期管理等核心功能。

## 5. 业务逻辑层 (Service Layer)

### 5.1 用户服务 (UsersService)

| 编号 | 任务 | 说明 |
|------|------|------|
| 5.1.1 | 用户注册 | 新增用户，保存用户信息 |
| 5.1.2 | 用户登录认证 | 用户名+密码校验，返回用户对象 |
| 5.1.3 | 用户信息查询 | 根据 ID 查询用户、查询所有用户 |
| 5.1.4 | 修改密码 | 校验旧密码，更新新密码 |
| 5.1.5 | 角色权限管理 | 区分普通用户(USER)和管理员(ADMIN)，id=1默认为管理员 |
| 5.1.6 | 个人信息管理 | 查看/修改用户名、邮箱，显示角色与注册时间 |

### 5.2 商品服务 (ProductService)

| 编号 | 任务 | 说明 |
|------|------|------|
| 5.2.1 | 商品新增 | 创建商品记录 |
| 5.2.2 | 商品查询 | 按 ID 查询、查询全部商品列表 |
| 5.2.3 | 商品更新 | 修改商品信息（名称、价格、描述、库存等） |
| 5.2.4 | 商品删除 | 根据 ID 删除商品 |

### 5.3 购物车服务 (CartService)

| 编号 | 任务 | 说明 |
|------|------|------|
| 5.3.1 | 添加购物车 | 将商品加入购物车，已有则增加数量 |
| 5.3.2 | 查看购物车 | 获取用户购物车列表（按时间倒序） |
| 5.3.3 | 修改数量 | 更新购物车中某商品的数量 |
| 5.3.4 | 移除商品 | 从购物车移除指定商品 |
| 5.3.5 | 清空购物车 | 清空用户整个购物车 |

### 5.4 订单服务 (OrderService)

| 编号 | 任务 | 说明 |
|------|------|------|
| 5.4.1 | 创建订单 | 从购物车选中商品创建订单，扣减库存，生成订单号，清除已购商品 |
| 5.4.2 | 查询用户订单列表 | 获取用户所有订单（按时间倒序） |
| 5.4.3 | 查询订单详情 | 根据订单号查询订单及订单项详情 |
| 5.4.4 | 订单支付 | 将订单状态从 PENDING → PAID，记录支付时间 |
| 5.4.5 | 取消订单 | 取消待支付订单，恢复库存，状态改为 CANCELLED |
| 5.4.6 | 查询所有订单（管理） | 获取全部用户的订单列表（后台管理用） |

## 6. 接口控制层 (Controller Layer)

### 6.1 登录认证接口 (LoginController)

| 编号 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 6.1.1 | `/login` | GET | 跳转登录页面 |
| 6.1.2 | `/login` | POST | 用户登录，Session 存储用户信息 |
| 6.1.3 | `/logout` | GET | 退出登录，销毁 Session |
| 6.1.4 | `/api/currentUser` | GET | 获取当前登录用户信息 |
| 6.1.5 | `/api/changePassword` | POST | 修改密码 |
| 6.1.6 | `/api/profile` | GET | 获取当前用户个人信息（用户名、邮箱、角色、注册时间） |
| 6.1.7 | `/api/updateProfile` | POST | 修改个人信息（用户名、邮箱） |

### 6.2 注册接口 (RegistrationController)

| 编号 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 6.2.1 | `/register` | GET | 跳转注册页面 |
| 6.2.2 | `/register` | POST | 提交注册信息，创建用户 |

### 6.3 商品接口 (ProductController)

| 编号 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 6.3.1 | `/api/product/list` | GET | 获取所有商品列表 |
| 6.3.2 | `/api/product/upload` | POST | 上传商品（含图片，Multipart） |
| 6.3.3 | `/api/product/update` | PUT | 更新商品信息及图片 |
| 6.3.4 | `/api/product/delete/{id}` | DELETE | 删除指定商品 |

### 6.4 购物车接口 (CartController)

| 编号 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 6.4.1 | `/api/cart/add` | POST | 添加商品到购物车 |
| 6.4.2 | `/api/cart/list` | GET | 获取购物车列表 |
| 6.4.3 | `/api/cart/update` | PUT | 更新购物车商品数量 |
| 6.4.4 | `/api/cart/remove` | DELETE | 移除购物车中的商品 |
| 6.4.5 | `/api/cart/clear` | DELETE | 清空购物车 |

### 6.5 订单接口 (OrderController)

| 编号 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 6.5.1 | `/api/order/create` | POST | 创建订单（从购物车结算） |
| 6.5.2 | `/api/order/list` | GET | 获取用户订单列表 |
| 6.5.3 | `/api/order/detail` | GET | 查询订单详情（按订单号） |
| 6.5.4 | `/api/order/pay` | POST | 订单支付 |
| 6.5.5 | `/api/order/cancel` | POST | 取消订单 |
| 6.5.6 | `/api/order/all` | GET | 获取所有用户订单（后台管理） |

## 8. 核心功能模块汇总 (Feature Modules)

### 8.1 用户管理模块
```
8.1.1 用户注册
8.1.2 用户登录 / 登出（Session 管理）
8.1.3 获取当前用户信息
8.1.4 修改密码

8.1.6 个人信息管理（查看/修改用户名、邮箱、显示角色权限）
```

### 8.2 商品管理模块
```
8.2.1 商品列表展示（前台）
8.2.2 商品搜索
8.2.3 商品新增（含图片上传，MIME 校验，UUID 命名）
8.2.4 商品编辑更新
8.2.5 商品删除
8.2.6 商品图片静态资源映射（/uploads/）
```

### 8.3 购物车模块
```
8.3.1 添加商品到购物车（需登录）
8.3.2 查看购物车列表
8.3.3 修改商品数量（+/- 按钮及手动输入）
8.3.4 移除单个商品
8.3.5 清空购物车
8.3.6 多选 / 全选结算
8.3.7 价格合计实时计算
```

### 8.4 订单模块
```
8.4.1 购物车结算创建订单（库存校验 + 扣减）
8.4.2 订单号生成（时间戳 + 4位随机数）
8.4.3 订单列表查看
8.4.4 订单详情查看
8.4.5 订单支付（PENDING → PAID）
8.4.6 订单取消（PENDING → CANCELLED，恢复库存）
8.4.7 订单状态流转：PENDING → PAID → SHIPPED → COMPLETED / CANCELLED
8.4.8 后台查看所有用户订单
```
